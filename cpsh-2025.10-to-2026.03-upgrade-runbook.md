# Collibra CPSH 2025.10 to 2026.03 Upgrade Runbook

This runbook covers upgrading Collibra Platform Self-Hosted (CPSH) from 2025.10 to 2026.03, including the required PostgreSQL 14 to PostgreSQL 17 transition, followed by upgrading CPSH Edge through the CPSH Extended Capabilities Zarf package.

## Target Versions

- CPSH installer: `dgc-linux-2026.03.353.sh`
- Required PostgreSQL version: PostgreSQL 17
- Previous PostgreSQL version: PostgreSQL 14
- Edge/Zarf package bundle: `zarf-packages-amd64-1.51.11.tar.gz`
- Edge package: `zarf-package-cpsh-edge-amd64-1.51.11.tar.zst`

Important: Use the latest Edge package that Collibra publishes as compatible with CPSH 2026.03. Do not assume the newest Edge package line is compatible unless it appears in Collibra's compatibility matrix for CPSH 2026.03.

## 1. Pre-Upgrade Checks

Run these checks on each CPSH node.

```bash
sudo -i
df -h
free -h
cat /etc/os-release
getenforce 2>/dev/null || true
/opt/collibra/jre/bin/java -version 2>/dev/null || true
ls -ld /opt/collibra /opt/collibra_data
```

Check the current PostgreSQL 14 installation:

```bash
ls -ld /usr/pgsql-14 2>/dev/null
/usr/pgsql-14/bin/postgres --version 2>/dev/null
```

Install PostgreSQL 17 before running the CPSH 2026.03 installer. Keep PostgreSQL 14 installed until the CPSH upgrade is complete and validated.

If `/usr/pgsql-17` is missing and the node can reach an internal package mirror or the internet, install PostgreSQL 17:

```bash
sudo dnf install -y dnf-plugins-core

sudo dnf install -y \
  https://download.postgresql.org/pub/repos/yum/reporpms/EL-$(rpm -E %{rhel})-x86_64/pgdg-redhat-repo-latest.noarch.rpm

sudo dnf -qy module disable postgresql

sudo dnf config-manager --set-disabled pgdg18 || true
sudo dnf config-manager --set-enabled pgdg17 || true

sudo dnf clean all
sudo dnf install -y postgresql17 postgresql17-server postgresql17-contrib
```

If the node is air-gapped, create or request an offline PostgreSQL 17 RPM bundle using the same pattern as the PostgreSQL 14 air-gap bundle. The bundle must include these packages and dependencies for the exact target OS major version and architecture:

```text
postgresql17
postgresql17-server
postgresql17-contrib
```

After installing PostgreSQL 17, verify:

```bash
ls -ld /usr/pgsql-17
/usr/pgsql-17/bin/postgres --version
```

Expected result:

```text
postgres (PostgreSQL) 17.x
```

For FIPS-enabled CPSH, confirm the OS is in FIPS mode:

```bash
cat /proc/sys/crypto/fips_enabled
```

Expected result:

```text
1
```

## 2. Back Up CPSH

Take a VM snapshot before the upgrade if your platform supports it.

Then stop the Collibra environment from the Management Console:

```text
Console -> Infrastructure -> Default environment -> Stop
```

Wait until all services are stopped.

Stop Console and Agent:

```bash
sudo -i
systemctl stop collibra-agent
systemctl stop collibra-console
```

Create a backup directory:

```bash
mkdir -p /backup/cpsh-2026.03-upgrade-$(date +%Y%m%d)
```

Optional filesystem backup if enough disk space is available:

```bash
tar --xattrs --acls -cpf /backup/cpsh-2026.03-upgrade-$(date +%Y%m%d)/collibra-opt-and-data.tar \
  /opt/collibra /opt/collibra_data
```

## 3. Upgrade CPSH and Repository Database Runtime

Make the installer executable:

```bash
cd /tmp
chmod +x dgc-linux-2026.03.353.sh
```

For non-FIPS installations:

```bash
./dgc-linux-2026.03.353.sh
```

For FIPS installations:

```bash
./dgc-linux-2026.03.353.sh -- --fips
```

Answer the prompts as follows:

```text
Specify the installation directory [/opt/collibra]:
/opt/collibra contains a previous installation. Do you want to perform an update? [y/N]
y
Before you can schedule an upgrade, you need to:
- Create a backup of the entire environment.
- In Collibra Console, stop all running services on this node.
Have you completed these steps? [yes,NO]
yes
Specify the PostgreSQL 17 path [/usr/pgsql-17]:
/usr/pgsql-17
```

Let the installer complete. The installer should handle the CPSH upgrade and move the platform to PostgreSQL 17. Do not manually run `pg_upgrade` unless Collibra Support explicitly instructs you to do so.

For multi-node environments, run the same CPSH 2026.03 installer on each node before starting the environment. Do not start services until every CPSH node has been upgraded.

## 4. Validate CPSH Services

After the installer completes:

```bash
systemctl status collibra-console --no-pager
systemctl status collibra-agent --no-pager
```

Start the environment from the Management Console:

```text
Console -> Infrastructure -> Default environment -> Start
```

Validate listening ports:

```bash
ss -ltnp | egrep '5404|5443|4403|4420|4414'
```

Confirm in the UI:

```text
Repository is green
Data Governance Center is green
Search is green
Jobserver is green
Management Console shows 2026.03.353
```

After startup, run a Collibra reindex from the Console.

## 5. Back Up Edge

Run these checks on the Edge/Zarf host:

```bash
kubectl get nodes
kubectl get pods -A
kubectl get ns | grep -i edge
```

The namespace is often:

```text
collibra-edge
```

Back up the Edge site resources:

```bash
mkdir -p /backup/cpsh-2026.03-upgrade-$(date +%Y%m%d)

kubectl get -n collibra-edge secrets -l edge.collibra.com/backup -o yaml \
  > /backup/cpsh-2026.03-upgrade-$(date +%Y%m%d)/edge-backup.yaml
```

## 6. Prepare Zarf Packages

Extract the Zarf package bundle if needed:

```bash
cd /tmp
tar -xzf zarf-packages-amd64-1.51.11.tar.gz
```

Verify Zarf:

```bash
zarf version
```

Confirm cluster access:

```bash
kubectl get nodes
kubectl get pods -A
```

## 7. Upgrade Edge

Deploy the Edge Zarf package:

```bash
zarf package deploy /tmp/zarf-package-cpsh-edge-amd64-1.51.11.tar.zst
```

If the package requires variables, use the same values from the previous Edge deployment. The most important values are usually:

```text
PLATFORM_ID=https://<cpsh-host>:5443
SITE_NAME=<existing-edge-site-name>
```

Use the existing Edge site name. If the site name changes, Collibra may register a new Edge site instead of upgrading the existing one.

Watch the rollout:

```bash
kubectl get pods -A
kubectl get pods -n collibra-edge -w
```

## 8. Final Validation

Validate CPSH:

```bash
systemctl status collibra-console --no-pager
systemctl status collibra-agent --no-pager
```

Validate Edge:

```bash
kubectl get nodes
kubectl get pods -A
kubectl get pods -n collibra-edge
```

Validate URLs:

```text
https://<cpsh-host>:5404
https://<cpsh-host>:5443
```

Confirm in Collibra:

```text
DGC login works
Management Console login works
Repository is running and healthy
Search is running and healthy
Jobserver is running and healthy
Edge site shows healthy
Edge capabilities still exist
Connections still work
```

## 9. Rollback Notes

Rollback should use the VM snapshot or full filesystem/database backup taken before the upgrade.

Do not attempt to downgrade PostgreSQL 17 data files back to PostgreSQL 14 manually. If rollback is required, restore the pre-upgrade VM snapshot or restore the full pre-upgrade backup set.
